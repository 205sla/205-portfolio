개발 일지

처음으로 본격적으로 개발한 유니티 게임이라 시행착오가 많았다. 주요 트러블슈팅과 회고를 시간 순으로 기록한다.

오프닝 애니메이션

타이틀 화면에서 카드가 보드로 흩뿌려지는 시퀀스를 구현하는 과정. Coroutine으로 카드 한 장당 0.04초씩 딜레이를 주어 자연스러운 흐름을 만들었다.

💭
처음에는 Update() 안에서 처리했는데, 프레임 드랍이 심해 코루틴으로 바꿨다. 첫 코루틴이라 어색했지만 이후 거의 모든 비동기 처리를 코루틴으로 풀게 됐다.

카드 이동

드래그 시 손에서 보드로 카드가 이동하는 로직. 보드의 셀 좌표를 정수 그리드(0~7)로 두고, 카드의 월드 좌표를 셀 인덱스로 변환했다.

C# CardController.cs 
// 드래그 종료 시 가장 가까운 셀로 스냅
public void OnEndDrag(PointerEventData e) {
    Vector2Int cell = Board.WorldToCell(transform.position);
    if (Board.IsValid(cell) && canPlay) {
        PlayCard(cell);
    } else {
        ReturnToHand();
    }
}

JSON 데이터 읽기 오류

릴리스 직전 발견한, 유저 데이터 마이그레이션이 실패하던 버그. 앱 빌드에는 멀쩡했지만 실 기기에서만 재현됐다.

500줄짜리 카드 매니저 함수 (펼치기)CardManager.cs · 524 lines
C# CardManager.cs 
public CardData LoadCard(string id) {
    if (string.IsNullOrEmpty(id)) return null;
    var path = "cards/" + id;
    var json = Resources.Load<TextAsset>(path);
    if (json == null) {
        Debug.LogWarning($"Card not found: {id}");
        return null;
    }
    try {
        var data = JsonUtility.FromJson<CardData>(json.text);
        data.Validate();
        return data;
    } catch (Exception ex) {
        Debug.LogError(ex);
        return null;
    }
}
// ... (508 more lines)

핵심 원인은 Resources.Load가 IL2CPP 빌드에서 대소문자를 가렸던 것. PC 에디터에서는 무시되던 차이가 안드로이드 빌드에서만 발생했다.
